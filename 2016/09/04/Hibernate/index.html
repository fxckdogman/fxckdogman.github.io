Hibernate

一款ORM(Object ralation mapping：对象关系映射)的Dao层框架。操作数据库时，完全以面向对象的方式，不需要书写SQL语句。

框架搭建：

1. 导包。

lib文件夹下的required（必须）下jar包已经连接数据库需要的jar包/或Maven的pom文件中导入相关约束。

2. 创建数据库，准备实体类。

3.书写orm元数据

<class> package属性设置后  标签内其他需要使用到全包名的地方可以写简单类名
属性配置实体与表的对应关系   name（完整类目） table 数据库名
<id> name主键   <generator>主键生成策略 native
<property>除id之外的普通属性映射   如果实体中属性名和表中字段一致。column可以省略

4. 书写主配置文件

必选5个，数据库连接和方言
可选 打印 格式化sql 自动建表
引入元数据

API

Session是用来与数据库进行交互

对对象的save()update()delect()…等操作其实是改变对象的状态
1.瞬时态 (有id但是没有和session建立关系)
2.持久态 （有id和session建立关系）
3.游离态（没id没和session建立关系）

一级缓存

目的：提高效率
方法 ：查询会将resultSet集合中的内容放入缓存和快照中
下次查询时会先从缓存中查看
更改时会对比缓存中的快照，有变化同步到数据库

事务

特性：原子，一致，隔离，持久。

原则：业务开启之前开启事务，业务提交之后提交事务，如果执行过程中出现异常，回滚事务
Dao层需要session保存操作
Service层需要session控制事务
要确保使用的是一个session对象
解决方案：将session绑定在当前线程
Hibernate中获取session有getCurrentSession();
要使用这个方法，必须在主配置中有一段配置

thread

查询语言

HQL适用于多表不复杂时
Query query = session.createQuery(hql);
query.setLong();

query.setParameter();
query.setFirstResult();
query.setMaxResult();

结果：query.list();
query.uniqueResult();
Creteria 适用于单表条件查询（无语句面向对象查询）
获得：Creteria creteria = session.createCreteria(Customer.class);
creteria.list();
creteria.uniquerResult();
creteria.add(条件对象);
条件对象：Restrictions.eq(“cust_id”,1L);实体中主键名和实体中主键数据类型
count查询返回Long类型
离线查询
Web层凭空出现一个creteria
DetachedCriteria dc = DetachedCriteria(session);
Dao层 Criteria c = dc.getExecutableCriteria();
List list = c.list();
原生SQL

一对多

实体中使用<set>标签
Name：实体中集合的名称   column外键列名称 class与其关联的对象完整类名
<key>
<one-to-many>
级联操作：cascade  简化代码，在哪一方设置，将这方对象状态更新时同步到另一方对象
关系维护：inverse  性能优化（关系维护） 无论怎么放弃，都要有一方进行维护，一对多中，只能是一的一方放弃，因为字段在多的一方，必须维护自己的字段。
多对多
<set name  table>
    <key column>
        <many-to-many class column >
Name:当前对应的对象实体中集合的名称
Table中间表名
Column 当前对象引用当前对象的外键名称

Class 当前对象与那个对象是多对多关系
Clumn 当前对象对应引用别人的外键名

关系维护

必须有一方放弃维护

多表维护关系的手段就是向中间表插入数据，都维护会发生插入数据重复
级联操作
原理：
手动删除，是先将主表中的子表引用remove 将从表外键设置为null 然后删除
级联删除，就是将子表的外键引用设置为null然后删除主表子表

查询优化

类级别查询
Get方法 没有任何策略，调用即查询数据库加载数据
Load 方法 应用级别的加载策略
< class>中配置lazy false 懒加载
批量抓取batch-size：3
抓取数据时 一次性抓取几个的对象集合
