bos-项目底层代码构建

表现层代码抽取

首先要明确继承与实现的区别。（类与接口）Java-extends

思路：BaseAction继承ActionSupport。实现ModelDriven实现重写方法。子类直接继承获得页面传入的model对象。

例：

//获取当前类的类型。
public class BaseAction<T> extends ActionSupport implements ModelDriven<T> {
    //模型对象
    private T model;
    public T getModel() {
        return model;
    }

    //在构造方法中动态获取实体类型，通过反射创建model对象
    public BaseAction() {
        ParameterizedType genericSuperclass = (ParameterizedType) this.getClass().getGenericSuperclass();
        //获得BaseAction上声明的泛型数组
        Type[] actualTypeArguments = genericSuperclass.getActualTypeArguments();
        Class<T> entityClass = (Class<T>) actualTypeArguments[0];
        //通过反射创建对象
        try {
            model = entityClass.newInstance();
        } catch (InstantiationException e) {
            e.printStackTrace();
        } catch (IllegalAccessException e) {
            e.printStackTrace();
        }
    }
}

持久层代码抽取

思路：

将使用频率较高的一些方法封装至一个泛型接口IBaseDao中，在BaseDaoImpl泛型类中实现所有方法。
IUserDao接口通过实现IBaseDao接口，可以获得IBaseDao中所有的抽象方法且不必再次实现。
BaseDaoImpl实现类继承IBaseDao接口并实现接口中所有的方法，
这样BaseDaoImpl内部有常用方法的实现，任意一个类去继承即可获得所有的已实现的方法。
UserDao通过实现IUserDao接口可以获得IBaseDao中那些常用的方法以及自身需要实现的方法，再继承BaseDaoImpl获得要实现方法的实现。UserDao中就只需要实现IUserDao中另外扩展的方法了。

例：

public interface IBaseDao<T> {};内部定义常用的抽象方法。
public class BaseDaoImpl<T> implements IBaseDao<T>{};实现IBaseDao接口重写所有方法。
public interface IUserDao extends IBaseDao<User>{};IUserDao类继承IBaseDao获得抽象方法;
public class UserDaoImpl extends BaseDaoImpl<User> implements IUserDao {}
UserDaoImpl实现IUserDao获得父类IUserDao通过继承IBaseDao获得的常用抽象方法，又继承BaseDaoImpl获得常用方法的实现。

获得泛型类的方法

public Class<T> entityClass;
public BaseDaoImpl() {
    //获取父类类型的类文件
    ParameterizedType superclass = (ParameterizedType) this.getClass().getGenericSuperclass();
    //获取父类泛型括号中的泛型数组
    Type[] actualTypeArguments = superclass.getActualTypeArguments();
    entityClass = (Class<T>) actualTypeArguments[0];
}

好处：代码重用性、降低耦合性。
