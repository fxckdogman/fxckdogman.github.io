MyBatis

Mybatis是一个优秀的持久层框架，对jdbc的操作数据库过程进行了封装，开发者只需要关注SQL本身。
MyBatis通过xml或注解的方式将要执行的statement配置起来，通过java对象和statement中的sql进行映射生成最终执行的sql语句，最后由mybaties框架执行sql并结果映射成java对象并返回。

mybaties入门(传统方法)
1.创建java工程
2.导入jar包。
3.创建主配置文件SqlMapperConfig.xml(config/SqlMapperConfig.xml)
4.创建domain对象。
5.创建sql映射文件User.xml(config/User.xml)
6.将config下的User.xml添加到主配置文件中

映射文件编写

User.xml添加select标签编写sql.
<select namespace="" id="" parameterTtpe="" resultType="">
    namespace:命名空间，用于隔离sql
    id:statement的id或者叫做sql的id
    parameterType:声明输入参数的类型
    resultType=声明返回值的类型
</select>

测试程序
1.创建SqlSessionFactoryBuilder.
2.加载SqlMapConfig配置文件
3.创建SqlSessionFactory对象
4.创建SqlSession对象
5.执行SqlSession对象查询，获取结果。
6.释放资源

#{}和${}

#{}表示一个占位符，通常#{}可以实现向parameterStatement占位符中设置值。会自动进行java类型和jdbc类型转换。
可以接收简单的类型值或pojo属性值。
如果parameterType传输单个简单类型值。#{}中可以是value或其他名称。

${}表示拼接sql字符串，通过${}可以将parameterType忠臣传入的内容拼接在一起而不进行jdbc类型转换，${}可以接受简单类型值或pojo属性值。
如果parameterType传输的是单个简单类型值，${}值只能是value。

自增主键返回

<select>
    <selectKey keyColumn="" keyProperty="" order="" resultType="">
        order:执行语句前还是执行语句后执行查询id的sql
        int类型id用AFTER
        uuid 用BEFORE
        resultType：自增int型id用 int  uuid为String
        SELECT LAST_INSERT_ID()
    </selectKey>
    INSERT INTO user (username,sex,birthday,address) VALUES (${username},${sex},${birthday},${address})
</select>

SELECT LAST_INSERT_ID()是mysql的函数，返回auto_increment自增列新记录id.

parameterType&RusultType
1.parameterType：
指定输入参数类型，mybatis通过ognl(对象导航语言)从输入对象中获取参数值拼接在sql中
2.resultType:
指定输出结果类型，mybatis将sql查询结果的一行记录数据映射为resultType类型指定类型的对象。如果有多条数据，则分别进行映射，并把对象放在容器List中。

Dao开发方法

Mapper动态代理开发
只需要编写Mapper接口，MyBatis框架根据接口创建接口动态代理对象。
需要一个Mapper接口文件。(类似于Dao层)一个Mapper.xml配置文件。
需遵循的规范：
1.Mapper.xml文件中的namespace与接口类路径相同。
2.Mapper接口方法名和Mapper.xml中定义的每个statement的id相同。
3.Mapper接口方法的返回值类型和Mapper.xml中定义的每个sql的parameterType类型相同。
4.Mapper借口哦方法的输入参数类型和Mapper.xml中定义的每个sql的parameterType类型相同。

Mapper.xml配置文件内容。

约束：

<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">


<select id="mapper接口方法名一致" parameterType="与接口方法输入参数类型一致" resultType="与接口方法返回参数类型一致">
   sql语句
</select>

SqlMapConfig.xml

配置内容
SqlMapConfig.xml中配置的内容和顺序如下：
properties（属性）
settings（全局配置参数）
typeAliases（类型别名）
typeHandlers（类型处理器）
objectFactory（对象工厂）
plugins（插件）
environments（环境集合属性对象）
environment（环境子属性对象）
transactionManager（事务管理）
dataSource（数据源）
mappers（映射器）

properties属性

<properties resource="加载外部配置文件">
    typeAlias属性(类型别名 )
    <typeAliases>
    <!-- 自定义单个别名-->
    <typeAlias alias="user" type="cn.pronfreebook.mybatis.domain.User">
    <!--批量定义别名 扫描包下的所有类 默认别名为类名，大小写不敏感-->
    <package name="cn.pronfreebook.mybatis.demain">
</typeAliases>

mappers属性

<mappers>
    <!--加载单个配置文件-->
    <mapper resource="sqlmap/User.xml">
    <!--加载包下所有配置文件-->
    要求mapper接口名称和mapper映射文件名称相同，在同一个目录下。
    <package resource="cn.pronfreebook.mybatis.mapper">
</mappers>

resultMap
resultType可以指定将查询结果映射为pojo，但需要pojo的属性名和sql查询的列名一致方可映射成功。
如果sql查询字段名和pojo的属性名不一致，可以通过resultMap将字段名和属性名作一个对应关系 ，resultMap实质上还需要将查询结果映射到pojo对象中。
resultMap可以实现将查询结果映射为复杂类型的pojo，比如在查询结果映射对象中包括pojo和list实现一对一查询和一对多查询。

当出现查询的列(数据库字段)和实体类属性不一致时，查询出的结果不能映射到pojo中。
可以自定义resultMap，将其对应起来。

例子：

<mapper namespace="cn.itcast.mybatis.mapper.OrderMapper">
<!-- resultMap要将不对应的结果映射到pojo上，type指定映射到哪一个pojo -->
<!-- id：设置ResultMap的id -->
<resultMap type="order" id="orderResultMap">
    <!-- 定义主键 ,非常重要。如果是多个字段,则定义多个id -->
    <!-- property：主键是实体类中的属性名 -->
    <!-- column：主键在数据库中的字段名 -->
    <id property="id" column="id" />
    <!-- 定义普通属性 -->
    <result property="userId" column="user_id" />
    <result property="number" column="number" />
    <result property="createtime" column="createtime" />
    <result property="note" column="note" />
</resultMap>

动态sql

通过mybatis提供的标签实现动态拼接sql语句。
<if>标签：判断当前字段是否存在来实现是否添加查询条件。
<where>标签：自动添加where，同时处理sql语句中的第一个and关键字，即如果第一个<if>条件不满足，而第二个<if>满足条件，去除第二个<if>将要添加sql字段的AND关键字。

sql片段：将重复的sql提取出来，用<include> refid属性引用。
<sql id="xx">
    sql片段
</sql>

如果要使用别的Mapper.xml配置的sql片段，可以在refid前面加上对应的Mapper.xml的namespace。

foreach标签：

向sql传递数组或者List时使用。
<foreach collection="ids" item="item" open="id IN (" close=")" separator=",">
        #{item}
</foreach>
